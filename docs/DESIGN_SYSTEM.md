# Design System

## Purpose

This document defines the visual identity for the Ancient Human Relatives web quest and final PDF report.

The design should guide Codex when building:

* the web app
* the species browsing pages
* the guided web quest
* the report preview
* the final PDF

---

# 1. Overall Design Aim

The app should feel like a polished life sciences magazine mixed with a guided digital workbook.

The final PDF should feel like a science field report that pupils would be proud to show at home.

The project should not feel like:

* a plain worksheet
* a generic form
* a childish school website
* a basic PDF export
* a simple poster-making task

---

# 2. Visual Direction

The visual identity should combine:

* popular science documentary
* life sciences magazine
* museum exhibition guide
* fossil record / deep time
* cinematic human evolution imagery
* clean digital workbook usability

The look should be modern, image-led and structured.

---

# 3. Mood

The design should feel:

* scientific
* dramatic
* exploratory
* polished
* mature
* structured
* evidence-based
* accessible

Avoid making it too cartoonish or primary-school-like.

The poster images have a colourful illustrated science-video style. The app and PDF should use those assets but frame them within a more polished magazine/report layout.

---


## Suggested Use

### Cover / Hero Areas

Use:

* deep navy
* almost black
* warm gold
* cinematic gradients
* high contrast text
* large image backgrounds

### Workbook / Form Areas

Use:

* cream
* fossil beige
* white or soft-white cards
* navy text
* coloured section labels
* clear borders

### Evidence / Dossier Areas

Use:

* cream paper background
* dark headings
* stamped labels
* small gold/rust accents
* evidence-card styling

### Timeline Areas

Use either:

* dark background with coloured bars, or
* cream background with strong navy/gold labels

Do not make the timeline look like a straight ladder toward *Homo sapiens*.

---

# 5. Typography

## Heading Fonts

Use a bold, condensed or modern heading font.

Good options:

* Oswald
* Bebas Neue
* Anton
* Space Grotesk

## Body Fonts

Use a readable sans-serif font.

Good options:

* Inter
* Source Sans 3
* Atkinson Hyperlegible
* IBM Plex Sans

## Typography Rules

* Headings should be bold and punchy.
* Body text must be very readable.
* Avoid tiny text, especially in the PDF.
* Use short paragraphs.
* Use labels, cards and prompts rather than walls of text.
* Scientific names should be italicised where possible.
* Use consistent heading levels.

---

# 6. Component Style

The app should use:

* rounded cards
* image-led panels
* strong section headers
* fact badges
* evidence cards
* pull quotes
* progress indicators
* clear navigation buttons
* large species cards
* concise prompts
* answer boxes with character guidance

Avoid clutter. The design should look premium but not overloaded.

---

# 7. Web App Layout

The web app should include:

* a strong landing page
* a clear start/continue flow
* species cards in a responsive grid
* guided step-by-step task pages
* persistent progress indicator
* save status indicator
* report preview button
* reset button with confirmation

## Web Page Feel

The web quest pages should feel structured and calm. Students should always know:

* what section they are on
* what they need to do
* how much they have completed
* how to move forwards or backwards
* whether their work has saved

---

# 8. Landing Page

The landing page should use the cover image and establish the magazine/science-documentary tone.

It should include:

* title: **Ancient Human Relatives**
* subtitle: **A web quest into the branching human family tree**
* short explanation
* start button
* continue button if saved work exists
* optional teacher notes / sources link

The landing page should feel more like an exhibit or magazine feature than a school form.

---

# 9. Species Cards

Each species card should show:

* poster image
* number
* name
* date range
* short hook
* extension label if relevant

Cards should be image-led and visually appealing.

## Core Species

The eight core species/groups should appear as the main sequence:

1. *Australopithecus afarensis*
2. *Homo habilis*
3. *Homo erectus*
4. *Homo heidelbergensis*
5. Neanderthals
6. Denisovans
7. *Homo floresiensis*
8. *Homo sapiens*

## Extension Species

*Homo naledi* should appear as an extension card.

It should be visually connected to the set but labelled clearly as extension.

---

# 10. Guided Task Pages

Each guided task page should include:

* section title
* short explanation
* clear student prompt
* input area
* character guidance
* save status
* next/back navigation
* progress indicator

The app should avoid huge text blocks. Break tasks into manageable cards or panels.

---

# 11. Report Design

The PDF report should be designed as an 8-page magazine-style field report.

The report should not look like a web form printout.

## Fixed PDF Pages

The PDF should use these fixed report pages:

1. Cover page
2. Species feature
3. Evidence dossier
4. Life and adaptations
5. Comparison spread
6. Timeline and overlap
7. Final article
8. Reflection / back cover

Each page should have a distinct visual purpose.

---

# 12. Report Page Style

Each PDF page should:

* have a clear title
* use consistent margins
* use strong visual hierarchy
* have controlled page breaks
* use large visuals where appropriate
* include enough white space
* avoid text overflow
* use print-safe sizing

The PDF should look good both:

* on screen as a preview
* exported/downloaded as a PDF

---

# 13. Report Page Details

## Page 1: Cover

Visual style:

* full-page image or dark cinematic background
* large title
* subtitle
* student details in a clean label/panel
* minimal body text

Should feel like a magazine cover or documentary title card.

## Page 2: Species Feature

Visual style:

* large species poster
* fact cards
* pull quote
* date/location badges
* strong species title

Purpose:

Introduce the chosen species and make it visually exciting.

## Page 3: Evidence Dossier

Visual style:

* case-file feel
* evidence cards
* fossils/tools/DNA/archaeology sections
* “evidence reviewed” label
* uncertainty box

Purpose:

Show how scientists know what they know.

## Page 4: Life and Adaptations

Visual style:

* anatomy and environment focus
* adaptation cards
* survival pressure explanation
* cautious wording prompts

Purpose:

Explain biology, structure and survival.

## Page 5: Comparison Spread

Visual style:

* three-column comparison
* chosen group, *Homo sapiens*, comparison group
* profile cards or comparison table
* similarity/difference callouts

Purpose:

Make the branching-tree idea more concrete through comparison.

## Page 6: Timeline and Overlap

Visual style:

* visual timeline
* overlapping date bars
* callout boxes
* branching-tree message

Purpose:

Show that different human groups overlapped and did not form a simple ladder.

## Page 7: Final Article

Visual style:

* magazine article layout
* large title
* possible drop cap
* pull quote
* final judgement badge
* clean readable essay text

Purpose:

Present the final written evaluation as a serious piece of science writing.

## Page 8: Reflection / Back Cover

Visual style:

* back-cover feel
* lighter than the main article
* checklist
* reflection answers
* source/credit note
* strong final statement

Purpose:

Close the report and make the student reflect on their work.

---

# 14. Print and PDF Design Rules

The report preview should be built with print/PDF export in mind.

Use:

* fixed page containers
* controlled margins
* page-break rules
* print-specific CSS
* image aspect-ratio handling
* overflow protection
* readable font sizes

Avoid:

* content that spills unpredictably
* uncontrolled long text blocks
* tiny text
* awkward page breaks
* image cropping that loses important content

If a student leaves a section incomplete, the page should still render cleanly.

# 16. Accessibility

The app should:

* use strong contrast
* have readable text sizes
* support keyboard navigation
* use semantic headings
* include alt text for images
* not rely on colour alone
* make form labels clear
* avoid inaccessible hover-only controls
* use clear focus states

---

# 17. Responsive Design

The web quest should work on:

* school laptops
* desktop computers
* tablets

Mobile support is useful but not the main priority.

The PDF preview should be readable on screen but optimised for export.

---

# 18. Scientific Presentation

The design should avoid implying:

* evolution is a ladder
* *Homo sapiens* are the goal of evolution
* all species are direct ancestors of each other
* extinct species were failures
* bigger brains always mean greater success

Use visual structures that support a branching, overlapping model.

Good visual approaches:

* branching diagrams
* overlapping timeline bars
* comparison cards
* evidence dossiers
* uncertainty callouts

Avoid visual approaches that look like:

```txt
primitive → better → modern human
```

---

# 19. Language and Labels

Use cautious scientific labels where appropriate:

* “evidence suggests”
* “may have”
* “possibly”
* “scientists debate”
* “uncertain”
* “known from”
* “based on current evidence”

Avoid overconfident claims where the evidence is limited.

---

# 20. Overall Design Success Criteria

The design is successful if:

* pupils want to click through the activity
* the species cards look exciting
* the task pages feel clear and calm
* the report preview looks premium
* the exported PDF looks worth showing at home
* the design reinforces the branching-tree model of human evolution
* the app feels like a science publication, not a worksheet
